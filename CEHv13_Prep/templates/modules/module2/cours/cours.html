1. Introduction au Footprinting

Le Footprinting est la première étape de toute démarche de piratage éthique ou de test d'intrusion. Il s'agit d'une phase de collecte d'informations visant à obtenir le maximum de données sur une cible, qu'il s'agisse d'une entreprise, d'un serveur ou d'une personne, sans interagir directement avec le système cible. Cette phase est essentielle pour comprendre l'environnement de la cible et planifier les prochaines étapes de l'attaque ou de l'audit.

🎯 Objectifs du Footprinting

  • Identifier les systèmes, réseaux et technologies utilisés par la cible.
  • Recueillir des informations sur les employés, les sous-domaines, les IPs, les serveurs, etc.
  • Préparer les futures attaques en minimisant les risques d’être détecté.

📦 Types de Footprinting

  • Footprinting passif : collecte d'informations sans interagir avec la cible, par exemple via les moteurs de recherche, réseaux sociaux, archives Internet, etc.
  • Footprinting actif : implique une interaction directe avec la cible, comme un ping ou une requête DNS. Plus risqué, car il peut générer des logs.

🔍 Méthodes de Footprinting passif

  • Recherche sur Google avec des Google Dorks.
  • Analyse des métadonnées dans les documents PDF, Word, etc.
  • Consultation de bases de données publiques comme WHOIS.
  • Utilisation d’archives comme Wayback Machine.
  • Recherche sur les réseaux sociaux (LinkedIn, Facebook) pour identifier les employés et leurs rôles.

⚒️ Outils utilisés pour le Footprinting

  • Maltego : outil d’analyse de relations et de liens.
  • theHarvester : collecte des emails, noms de domaines et sous-domaines.
  • Recon-ng : framework de reconnaissance modulaire en ligne de commande.
  • Shodan : moteur de recherche d'appareils connectés à Internet.
  • Nslookup / host / dig : outils de requête DNS.

💡 Bonnes pratiques

  • Commencez toujours par le footprinting passif avant de passer à l'actif.
  • Documentez chaque information collectée pour l’utiliser dans les étapes suivantes (scanning, enumeration, etc.).
  • Respectez la légalité : le footprinting passif est souvent légal car basé sur des données publiques, mais l'actif peut ne pas l’être sans autorisation explicite.

En résumé, le Footprinting est une étape de reconnaissance discrète, stratégique et indispensable dans tout processus de test de sécurité. Une bonne phase de collecte peut faire la différence entre un test efficace et une perte de temps.

2. Email Footprinting

Le email footprinting (ou empreinte par courrier électronique) est une technique de reconnaissance consistant à obtenir des informations précieuses à partir des e-mails envoyés ou reçus. Ces informations peuvent être utilisées à des fins de veille, de ciblage marketing ou d’attaques par ingénierie sociale.

Suivi des emails

Le suivi des e-mails se fait généralement par l’analyse des en-têtes d’e-mail (email headers) et l’utilisation d’outils de traçage. Cela permet notamment d’obtenir :

  • L’adresse IP du destinataire
  • La géolocalisation approximative
  • Le statut de livraison du message
  • Les liens cliqués dans l’e-mail
  • Le navigateur et le système d’exploitation utilisés
  • La durée de lecture du message

Des outils de suivi des e-mails (email trackers) sont souvent utilisés par les commerciaux, les entreprises de marketing ou les attaquants. Ces outils peuvent notifier l’expéditeur lorsque l’e-mail est ouvert ou lorsqu’un lien est cliqué.

Analyse des en-têtes d’e-mail

L’analyse des en-têtes permet de déterminer si un e-mail est légitime ou potentiellement malveillant. Ces en-têtes contiennent :

  • Le nom et l’adresse IP de l’expéditeur
  • L’adresse e-mail d’origine
  • Le serveur de messagerie utilisé
  • Les horodatages d’envoi et de réception
  • Un identifiant unique de message

Protocoles d’authentification

Plusieurs protocoles sont intégrés dans les en-têtes pour authentifier l’expéditeur :

  • SPF (Sender Policy Framework) : Vérifie si le serveur émetteur est autorisé à envoyer des mails pour un domaine. Exemple : PASS avec l’IP 209.85.220.69.
  • DKIM (DomainKeys Identified Mail) : Utilise une signature numérique pour vérifier que le message provient bien du domaine d’origine. Exemple : PASS avec domain=google.com.
  • DMARC (Domain-based Message Authentication, Reporting and Conformance) : Combine SPF et DKIM pour renforcer la vérification et appliquer des politiques (refuser ou accepter).

Vérification de la légitimité d’un e-mail

  • Vérifiez soigneusement l’adresse FROM
  • Surveillez les fautes d’orthographe dans le domaine (ex: "g00gle.com")
  • Analysez l’adresse IP de l’expéditeur (est-elle publique ? privée ? domestique ?)
  • Vérifiez si l’adresse IP correspond bien au domaine indiqué

Certains fournisseurs comme Google masquent l’IP de l’expéditeur en la relayant via des serveurs internes, rendant la traçabilité difficile.

Politiques des fournisseurs d’e-mails

Chaque fournisseur de messagerie (Gmail, Outlook, Yahoo, etc.) applique ses propres règles sur les serveurs SMTP. Certains bloquent l’affichage de l’adresse IP, interdisent certains types de pièces jointes ou limitent l’envoi massif. Un attaquant peut étudier ces politiques pour adapter ses tactiques.

Récupération de l’adresse IP d’un utilisateur

Il existe des méthodes pour récupérer l’adresse IP d’un destinataire :

  • Inclure une image hébergée sur un serveur que vous contrôlez
  • Inclure un lien traquant vers une ressource (même invisible)
  • Utiliser l’ingénierie sociale pour inciter l’utilisateur à cliquer

Exemple d’exploitation :

  1. Installez Apache : sudo apt install apache2
  2. Démarrez le serveur : sudo systemctl start apache2
  3. Créez un fichier dans /var/www/html : touch image.jpg
  4. Surveillez les accès : tail -f /var/log/apache2/access.log
  5. Envoyez un e-mail contenant un lien vers http://votre_IP/image.jpg
  6. Récupérez l’adresse IP quand le fichier est chargé par le destinataire

Une fois l’IP obtenue, vous pouvez utiliser geoiplookup ou ipinfo.io pour obtenir sa géolocalisation.

Bonnes pratiques de protection

  • Désactiver l’affichage automatique des images dans les e-mails
  • Utiliser des fournisseurs de messagerie anonymes ou avec relais IP
  • Éviter de cliquer sur des liens suspects
  • Analyser les en-têtes d’e-mail avec des outils comme MXToolbox

3. Footprinting Réseau (Network Footprinting)

Le footprinting réseau est une étape essentielle du processus de reconnaissance dans un test d’intrusion. Il consiste à collecter des informations sur l’infrastructure réseau d’une cible afin de comprendre son architecture, les machines qui y sont connectées, et les services qu’elle expose. Cette phase est cruciale pour planifier efficacement les étapes suivantes d’un audit ou d'une attaque.

Objectifs du footprinting réseau

  • Identifier la plage d’adresses IP utilisée par la cible
  • Repérer les hôtes actifs et les services en écoute
  • Comprendre la structure du réseau
  • Déterminer les systèmes d’exploitation utilisés
  • Détecter la présence de pare-feu ou d’autres dispositifs de sécurité

Nmap - Outil de découverte réseau

Nmap (Network Mapper) est un outil incontournable pour la découverte de réseau. Il permet de scanner des adresses IP ou des plages entières pour identifier :

  • Les hôtes actifs
  • Les ports ouverts
  • Les services et leurs versions
  • Les systèmes d’exploitation (via fingerprinting)
  • Les types de pare-feux ou de filtrage en place

Nmap est utilisé aussi bien par les administrateurs réseau que par les attaquants. Il aide à détecter des failles, mais aussi à s’assurer que les configurations réseau sont correctes (par exemple, vérifier si un port est bien fermé). 

nmap -v -p 0-2000 -O -sV 178.128.203.1
Option Description
-vMode verbeux, plus de détails sur le scan
-dMode debug, encore plus de détails (utile pour le dépannage)
-pSpécifie les ports à scanner (ex. -p 80 ou -p 1-65535)
-PnDésactive la détection d’hôtes (utile si la cible ne répond pas au ping)
-snScan sans port (ping scan), juste pour détecter les hôtes en ligne
-sSScan SYN (stealth), envoie des paquets SYN pour détecter les ports ouverts sans établir de connexion complète
-sTScan TCP complet (connexion normale, plus détectable)
-sUScan UDP (utilisé pour les services DNS, SNMP, etc.)
-sVDétection des versions des services
-ODétection du système d’exploitation
-AScan agressif (inclut OS, version, traceroute, script par défaut)
-T4Ajuste la vitesse du scan (de T0 à T5) – T4 est rapide et fiable
-oNSortie normale vers un fichier (nmap -oN result.txt)
-oXSortie au format XML
-iLLire les cibles depuis un fichier texte
-oGSortie au format “grepable”, utile pour automatiser
--openAffiche uniquement les ports ouverts
--scriptLance des scripts NSE (ex. --script=http-title)

💡 Pour plus d’options, vous pouvez consulter la documentation complète sur le site officiel : https://nmap.org/book/man-briefoptions.html

💡 Grâce aux versions détectées, il est possible de rechercher des failles connues dans des bases de données comme Exploit-DB.

⚖️ Attention : dans plusieurs pays comme le Royaume-Uni et l'Allemagne, l'utilisation de Nmap sans autorisation est considérée comme illégale. Consultez la documentation officielle pour plus d’informations légales.

Traceroute - Analyse du chemin réseau

L’outil Traceroute permet d’identifier les routeurs intermédiaires entre votre machine et la cible. Cela permet de reconstituer le chemin que prend un paquet réseau et d’analyser la topologie du réseau.

  • Chaque saut (hop) représente un routeur intermédiaire
  • Il est possible de découvrir des routeurs de confiance, des pare-feux ou des segments réseau cachés
  • Certains sauts peuvent être invisibles ou volontairement filtrés

Traceroute utilise le champ TTL (Time To Live) des paquets IP :

  • Débute avec un TTL de 1 et l’incrémente à chaque saut
  • À chaque expiration du TTL, le routeur répond avec un message ICMP
  • On peut ainsi tracer chaque routeur sur le chemin vers la cible

Ces informations sont utiles pour créer des cartes réseau, contourner les protections, ou préparer des attaques comme le "Man-in-the-Middle".

Commandes utiles :

  • traceroute 178.128.203.1 (Unix, utilise UDP)
  • tracert 178.128.203.1 (Windows, utilise ICMP)
  • nmap --traceroute --script traceroute-geolocation 178.128.203.1 -d (Traceroute avec géolocalisation)
  • hping3 --traceroute -S [IP cible] (plus personnalisable, utilise TCP SYN)

🧠 Ces outils permettent également de repérer les équipements de sécurité, les points faibles de filtrage et d’optimiser les futures attaques ou audits de sécurité.

4. Reconnaissance via les moteurs de recherche et ressources en ligne

La phase de reconnaissance passive en cybersécurité consiste à collecter un maximum d'informations sur une cible sans interagir directement avec elle. Internet regorge de ressources pour obtenir ces données : employés, intranet, adresses e-mail, pages de connexion, fuites de données, infrastructures techniques, etc.

Sources d'information

  • Réseaux sociaux (LinkedIn, Facebook, Twitter…)
  • Services de recherche de personnes (Pipl, Hunter.io…)
  • Services d’alertes (Google Alerts…)
  • Sites d’emploi (souvent riches en détails sur l’infrastructure de l’entreprise)
  • Services financiers, annuaires, blogs
  • Web profond (Deep Web) et web obscur (Dark Web)

Google Hacking (Recherche avancée)

Le Google Hacking consiste à utiliser des opérateurs de recherche (aussi appelés Google Dorks) pour formuler des requêtes complexes permettant de découvrir des informations sensibles accessibles publiquement.

💡 Pour faciliter les recherches complexes, utilisez :

  • Google Advanced Search
  • Google Advanced Image Search

Outils de Google Hacking

  • Google Hack Honeypot : détecte les requêtes malveillantes envoyées à votre site
  • Google Hacking Database (GHDB) : liste de requêtes utiles pour trouver des informations sensibles

Metagoofil

Metagoofil est un outil open source qui extrait les métadonnées des documents publics disponibles en ligne (PDF, DOC, PPT…). Il fonctionne ainsi :

  1. Recherche des documents à l’aide des Dorks
  2. Télécharge les fichiers
  3. Analyse les métadonnées (noms d’utilisateurs, logiciels utilisés…)

Services d'analyse en ligne

  • Website Watcher : alerte en cas de modification d’une page web
  • Recherche d’image inversée : Google Images, TinEye
  • Moteurs vidéo : YouTube, Google Vidéos, outils comme YouTube DataViewer, EZGif
  • Métamoteurs : Startpage, MetaGer (recherche confidentielle)
  • Moteurs FTP : Napalm FTP Indexer, Global FTP Search (souvent utilisé pour trouver des documents non protégés)

Recherche dans l’IoT

Ces moteurs de recherche permettent de trouver des appareils connectés à Internet (webcams, routeurs, serveurs…).

  • Shodan : moteur le plus connu
  • Censys : identifie les ressources Internet accessibles
  • Thingful : autre moteur de l’IoT

Outils supplémentaires

  • Netcraft : fournit des informations sur les technologies utilisées par un site
  • CrimeFlare : révèle les adresses IP réelles derrière Cloudflare

💬 Conseil : Toutes ces données doivent être croisées avec d’autres méthodes de collecte pour améliorer la précision. Les données récupérées via les moteurs peuvent être obsolètes ou inexactes.

5. Empreinte numérique d'un site web (Website Footprinting)

Le footprinting d'un site web est une technique utilisée pour recueillir discrètement des informations sur un site cible. Elle permet de dresser une carte complète de la structure et des technologies d’un site sans alerter le propriétaire. Ces données sont souvent utilisées lors des phases de reconnaissance dans un test d’intrusion ou par des cybercriminels pour préparer une attaque.

Informations obtenues

  • Technologies utilisées (logiciels, CMS, serveurs)
  • Système d'exploitation côté serveur
  • Répertoires et sous-répertoires accessibles
  • Informations de contact
  • Langages de programmation utilisés
  • Détails sur les requêtes et les scripts

1. Robots explorateurs (Web Spiders)

Les web spiders ou crawlers sont des programmes qui parcourent un site web de manière automatique pour extraire des informations. Ils sont souvent utilisés à des fins de référencement, mais peuvent aussi servir à collecter des données utiles à des attaques par ingénierie sociale.

2. Analyse des cookies

Les cookies peuvent révéler des informations sur les technologies utilisées par le serveur. Par exemple, un cookie nommé PHPSESSID indique l'utilisation de PHP. L'analyse des cookies permet aussi de comprendre le comportement de l'application web.

3. Analyse des en-têtes HTTP

Les en-têtes HTTP d’un site fournissent de précieuses informations :

  • Content-Type : Type de contenu (HTML, JSON, XML...)
  • Accept-Ranges : Support de la récupération partielle de fichiers
  • Last-Modified : Date de la dernière modification
  • X-Powered-By : Technologies utilisées (ex : ASP.NET, PHP, ZendServer...)
  • Server : Logiciel serveur et OS (ex : Apache sur CentOS)

Pour inspecter ces en-têtes, vous pouvez utiliser les outils développeur du navigateur (F12 ou Ctrl+Shift+C), puis consulter l’onglet Réseau (Network).

4. Analyse du code source

Commentaires HTML/JS

Le code source d’une page peut contenir des commentaires laissés par les développeurs. Ces commentaires, bien que non interprétés par le navigateur, peuvent révéler des chemins internes, des technologies utilisées ou même des failles connues.

Balises de liens et d'images

L’analyse des balises <a href> ou <img src> permet d’identifier la structure du site, les chemins vers les ressources, ou l'utilisation éventuelle de serveurs tiers (CDN, cache...).

5. Clonage de site (Website Mirroring)

Le clonage d’un site permet de le sauvegarder pour une analyse hors ligne, détecter des vulnérabilités, ou même créer un faux site (phishing). Cela peut être détecté par les systèmes de sécurité si trop rapide ou répétitif.

Outils utilisés

  • HTTrack : httrack https://sitecible.com
  • wget : wget -m https://sitecible.com
  • Copie manuelle du code source HTML + CSS

Conseils de sécurité pour les utilisateurs

  • Vérifiez l’URL : attention aux fautes d’orthographe
  • Assurez-vous que le site utilise le protocole HTTPS
  • Contrôlez les certificats SSL et leur validité
  • Méfiez-vous si le certificat change soudainement

6. Extraction de métadonnées

Les fichiers présents sur un site (PDF, images, documents...) contiennent souvent des métadonnées utiles :

  • Auteur du fichier
  • Logiciel utilisé pour le créer
  • Coordonnées GPS (images prises avec smartphone)

Outils d’extraction

  • ExifTool : exiftool fichier.jpg
  • Hexdump : Pour l'analyse binaire basique (hexdump -C fichier)
  • Metagoofil : Utilise Google pour rechercher des fichiers indexés et en extraire les métadonnées

💡 À savoir

  • Linux se base sur le type MIME réel du fichier, et non sur son extension
  • Les commentaires dans le code peuvent révéler plus que vous ne l’imaginez
  • Un site mal configuré peut exposer des répertoires sensibles (comme /admin, /backup, etc.)

Conclusion

Le footprinting est une étape critique pour comprendre l’architecture, la surface d’attaque, et les technologies d’un site web cible. Utilisé à bon escient, il permet d’anticiper les vulnérabilités et de renforcer la sécurité. Il est toutefois essentiel de respecter l’éthique et la légalité lors de la collecte de ces informations.

6. Reconnaissance : WHOIS, Géolocalisation IP et Interrogation DNS

Dans le cadre de la phase de reconnaissance passive d'une cible, l'analyse des informations publiques peut fournir de précieuses données, sans alerter la victime. WHOIS, la géolocalisation IP et l'interrogation DNS sont des outils essentiels à cette étape.

WHOIS : Recherche d’informations sur les domaines

WHOIS est un protocole (port 43) qui permet d’interroger les bases de données des noms de domaine. Les informations retournées sont publiques, leur consultation est légale.

  • Nom de domaine, date de création, expiration, dernière mise à jour
  • Détails sur le propriétaire (souvent protégés par un WHOIS guard)
  • Fournisseur d’enregistrement (ex : Gandi, NameCheap)
  • Serveurs DNS utilisés
  • Plage d’adresses IP associée

Exemple de commande : whois exemple.com ou via des sites comme whois.net.

Modèles de WHOIS

  • Thick WHOIS : les données sont centralisées, toutes les informations sont accessibles à partir d'une seule source.
  • Thin WHOIS : seules les données minimales sont disponibles, avec redirection vers d'autres serveurs pour plus d'informations.

Organismes régionaux (RIRs)

Les bases WHOIS sont gérées par des registres régionaux :

  • AFRINIC : Afrique
  • ARIN : Amérique du Nord
  • RIPE NCC : Europe, Moyen-Orient
  • APNIC : Asie-Pacifique
  • LACNIC : Amérique latine et Caraïbes

Géolocalisation IP

Grâce à des outils de géolocalisation, il est possible d’estimer l’emplacement géographique d’une adresse IP (pays, ville, FAI...).

Bien que les données de ville soient parfois approximatives, cela peut suffire à orienter une attaque d’ingénierie sociale. Exemple d’outil : GeoIPTool.com.

Interrogation DNS (Domain Name System)

Les requêtes DNS permettent d’obtenir des informations sur les noms de domaine, les adresses IP associées et les services liés (comme les e-mails).

Commandes utiles

  • host -t a exemple.com : récupère les enregistrements A (adresses IP)
  • host 192.168.0.1 : recherche DNS inverse (reverse DNS)
  • host -t mx exemple.com : identifie les serveurs de messagerie (MX)

Types d’enregistrements DNS

  • A : adresse IPv4 du domaine
  • AAAA : adresse IPv6
  • MX : serveurs de messagerie
  • CNAME : alias d’un domaine
  • NS : serveurs de noms
  • TXT : informations diverses (ex : SPF, vérification Google, etc.)

Utilité pour un attaquant

  • Identifier les fournisseurs de services (hébergement, messagerie...)
  • Repérer des points faibles (anciens serveurs, configurations mal sécurisées...)
  • Préparer des campagnes d’hameçonnage ciblé (phishing)
  • Observer les changements DNS (en lien avec un rachat, une migration, etc.)

Bonnes pratiques de défense

  • Utiliser un service WHOIS Guard pour masquer les données personnelles
  • Surveiller les enregistrements DNS avec des outils d’alerte
  • Limiter les données sensibles dans les enregistrements TXT
  • Mettre à jour régulièrement ses enregistrements DNS

Remarque : ces techniques sont aussi utilisées par les professionnels de la sécurité pour réaliser des audits ou tester la surface d’exposition publique d’une entreprise.
