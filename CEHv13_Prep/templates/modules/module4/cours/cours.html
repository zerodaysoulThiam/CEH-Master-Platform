1. Énumération DNS

Qu'est-ce que le DNS ?

Le Domain Name System (DNS) est un système hiérarchique et décentralisé permettant de traduire les noms de domaine (comme google.com) en adresses IP (comme 142.250.179.206). Il facilite la navigation sur Internet en permettant aux utilisateurs d’utiliser des noms faciles à retenir plutôt que des adresses numériques.

  • Utilise les ports UDP 53 (requêtes) et TCP 53 (transferts de zones).
  • Chaque domaine est géré par un ou plusieurs serveurs DNS contenant des fichiers de zone avec les enregistrements DNS.

Types d'enregistrements DNS

Les enregistrements DNS sont des entrées dans les fichiers de zone qui permettent d’associer des noms de domaine à diverses ressources.

  • A : associe un domaine à une adresse IPv4.
  • AAAA : associe un domaine à une adresse IPv6.
  • MX : enregistrements de messagerie (Mail eXchange) pour diriger les emails.
  • NS : désigne les serveurs de noms pour un domaine.
  • SOA : fournit des informations sur la zone, comme le numéro de série, le TTL, l’adresse e-mail du responsable, etc.
  • CNAME : alias vers un autre nom de domaine.
  • PTR : résolution inverse (IP → nom de domaine), souvent utilisée pour la vérification anti-spam.
  • HINFO : donne des informations sur le système (CPU et OS).
  • SRV : enregistrements de services (comme Kerberos, LDAP).

Techniques d'énumération DNS

L’énumération DNS est une étape clé du renseignement (reconnaissance passive/active). Elle permet d'obtenir des informations sensibles sur une organisation via son infrastructure DNS.

  • Récupération des enregistrements NS pour détecter les serveurs autoritaires.
  • Transferts de zone (zone transfers) pour aspirer la totalité d’une zone DNS (si mal configurée).
  • Bruteforce de sous-domaines à l’aide de dictionnaires (avec des outils comme dnsrecon ou dnscan).
  • Énumération des enregistrements SRV pour repérer les services réseau actifs.
  • Résolution inversée PTR sur des plages IP (CIDR).
  • Snooping du cache DNS pour vérifier quels noms de domaine ont été résolus récemment.

Snooping du cache DNS

Cette technique consiste à effectuer des requêtes DNS non récursives afin de savoir si un serveur DNS a déjà résolu un domaine. Cela peut révéler des sites fréquemment consultés par une organisation.

  • Utilise l’option +norecurse avec dig, ou -norecurse avec nslookup.
  • Permet d’identifier si un domaine est présent dans le cache DNS cible.

Transferts de zone DNS

Un transfert de zone DNS consiste à copier les enregistrements d’un serveur DNS vers un autre. Utilisé normalement entre serveurs maîtres et esclaves, il peut être abusé si mal sécurisé.

  • Utilise le protocole AXFR (transfert complet) ou IXFR (incrémental).
  • Un attaquant peut simuler un serveur esclave et demander un transfert de zone.
  • Ce type d’attaque révèle souvent des sous-domaines, des adresses internes ou des services actifs.

Exemple d’attaque par transfert de zone

  1. Obtenir les enregistrements NS : 
    dig ns example.com
  2. Essayer un transfert de zone : 
    dig axfr @ns1.example.com example.com

Contre-mesures

  • Restreindre les transferts de zone aux IP autorisées uniquement.
  • Utiliser un DNS split-horizon (ou split DNS) :
    • Fournir des réponses différentes selon l’origine de la requête (interne/externe).
    • Masque les ressources internes à Internet.

Outils d'énumération DNS

  • dnsrecon : outil Python open-source, très complet pour récupérer des enregistrements, tester les transferts de zone, faire du brute-force... 
    dnsrecon -d example.com
  • dig : outil UNIX pour requêtes DNS. 
    dig example.com any
dig axfr @ns1.example.com example.com
  • nslookup : permet des requêtes DNS interactives ou en ligne de commande. Moins flexible que dig. 
    nslookup
> server ns1.example.com
> set type=any
> example.com
  • host : outil Unix simple et rapide. 
    host -t ns example.com
host -l example.com

Remarques finales

Les informations obtenues par l’énumération DNS peuvent être cruciales pour une phase de reconnaissance dans un test d’intrusion. Bien configurés, les serveurs DNS doivent limiter ces possibilités, car même de simples erreurs de configuration peuvent exposer toute l’infrastructure réseau.

2. Vue d'ensemble de l'énumération

L’énumération est une phase cruciale du pentest : elle consiste à extraire des informations sensibles sur le système cible. On peut récupérer :

  • Tables de routage
  • Listes d'utilisateurs et groupes
  • Ressources réseau (partages, imprimantes...)
  • Informations de service ou de version
Elle s’appuie souvent sur les services détectés lors de la phase de scan (ex : SMB, NetBIOS, SNMP…).

🧠 Astuces et techniques de base

  • Créer un nom d'utilisateur à partir d'une adresse email : prenom.nom@exemple.com → prenom.nom
  • Essayer les mots de passe par défaut (admin:admin, root:root...)
  • Faire de l'énumération DNS (sous-domaines, enregistrements MX, etc.)

🪟 Énumération Windows

  • net share : affiche les partages locaux
  • net config : montre les infos réseau du système
  • net view \\machine : affiche les partages d’une machine distante
  • net use \\machine\ipc$ "" /user:"" : crée une session nulle
  • Utiliser des outils comme Hyena et DumpSec pour voir tous les comptes

🆔 SID (Security Identifier)

Windows attribue un identifiant unique à chaque compte : S-1-5-21-1852694824-1489621752-332472329-500
Le dernier chiffre est appelé RID :

  • 500 → Administrateur
  • 501 → Guest

🛠️ Outils pour SID et utilisateurs

  • user2sid / sid2user
  • Dumpusers
  • GetAcct
  • DumpSec
  • Hyena
  • PsGetSid

📡 Énumération NetBIOS / SMB

  • nbtstat -a IP : affiche les noms NetBIOS d'une cible
  • net view \\IP : liste les partages sur la cible
  • nat -o result.txt -u users.txt -p pass.txt 192.168.1.10 : test via SMB
  • Outil graphique : WinFingerprint

⚠️ Les failles SMB sont très critiques (ex : WannaCry exploitait SMBv1).

📊 Énumération SNMP

SNMP permet de gérer à distance des équipements réseau. Il repose sur :

  • OID : identifiants d’objet (ex. sysName.0)
  • MIB : structure de données décrivant les OID
Versions :
  • v1 & v2 : pas sécurisées (transfert clair)
  • v3 : chiffrée et authentifiée

🛠️ Outils SNMP

  • snmpwalk -c public -v1 192.168.1.10 : énumère tout
  • snmp-check 192.168.1.10 -v1 : infos utiles (OS, uptime, interfaces…)
  • snmpget -c public -v1 IP .1.3.6.1.2.1.1.5.0 : récupère sysName (nom système)
  • SNMP Walk (outil GUI Windows)

📚 Compléments et bonnes pratiques

  • Renseigne-toi sur les chaînes communautaires SNMP les plus fréquentes : public, private
  • Utilise des outils de brute-force ciblés : enum4linux, smbclient, ldapsearch
  • Pratique l'énumération "passive" avec Wireshark ou tcpdump quand tu n’as pas d'accès direct
