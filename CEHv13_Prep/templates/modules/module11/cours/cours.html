1. Concepts de Session Hijacking

Qu'est-ce que le Session Hijacking?

Le détournement de session est une technique où un attaquant prend le contrôle d'une session utilisateur légitime après l'authentification, en volant ou en devinant l'identifiant de session.

Pourquoi est-ce efficace?
  • Contourne l'authentification initiale
  • Permet un accès complet au compte
  • Difficile à détecter pour l'utilisateur
  • Peut être maintenu pendant longtemps

Types de sessions vulnérables

Type Description
Sessions web Cookies, tokens JWT, paramètres URL
Sessions réseau Connexions TCP, séquences de paquets
Sessions applicatives Authentification unique (SSO), API tokens

Impact du Session Hijacking

  • 68% des applications web sont vulnérables
  • Coût moyen d'une violation: $4.35 millions
  • 60% des attaques impliquent des identifiants volés
  • Temps moyen de détection: 207 jours

Cycle d'attaque typique

  1. Surveillance du trafic réseau
  2. Capture d'identifiants de session
  3. Prise de contrôle de la session
  4. Élévation de privilèges
  5. Exfiltration de données

2. Détournement au niveau applicatif

Méthodes d'attaque

Méthode Description
Vol de cookie XSS, MITM, accès physique
Fixation de session Forcer un ID de session spécifique
Prédiction d'ID Devinettes d'ID de session faibles
Cross-Site Request Forgery (CSRF) Exécution d'actions non autorisées

Exemple: Attaque par fixation de session

  1. Attaquant crée un ID de session valide
  2. Victime est dirigée vers le site avec cet ID
  3. Victime s'authentifie avec cet ID
  4. Attaquant utilise le même ID pour accéder au compte

Vecteurs d'attaque courants

  • XSS (Cross-Site Scripting): Exécution de scripts malveillants
  • MITM (Man-in-the-Middle): Interception du trafic
  • Sniffing WiFi: Capture de données sur réseaux non sécurisés
  • Applications compromises: Malware sur l'appareil de la victime

Démonstration: Vol de cookie via XSS

  1. Injecter un script dans une page vulnérable
  2. Script envoie les cookies de session à l'attaquant
  3. Attaquant copie les cookies dans son navigateur
  4. Accès au compte sans authentification

Protections applicatives

  • Cookies avec flags Secure et HttpOnly
  • Renouvellement d'ID de session après authentification
  • Validation des tokens CSRF
  • Mise en œuvre de Content Security Policy (CSP)

3. Détournement au niveau réseau

Attaques TCP/IP

Type Description
TCP Session Hijacking Prédiction des numéros de séquence
RST Attack Envoi de paquets RST pour terminer la session
Man-in-the-Middle (MITM) Interception complète du trafic
ARP Spoofing Redirection du trafic réseau

Processus de détournement TCP

  1. Surveillance de la connexion TCP
  2. Prédiction des numéros de séquence
  3. Envoi de paquets falsifiés
  4. Déconnexion de la victime (RST ou SYN flood)
  5. Prise de contrôle de la session

Exemple: ARP Spoofing

  1. Attaquant envoie de faux messages ARP
  2. Réseau redirige le trafic vers l'attaquant
  3. Attaquant intercepte et modifie les paquets
  4. Attaquant renvoie les paquets à destination
  5. Victimes inconscientes de la manipulation

Protocoles vulnérables

  • Telnet/FTP: Communications non chiffrées
  • HTTP: Trafic visible en clair
  • RDP: Sans sécurisation appropriée
  • SMTP/POP3: Authentification en clair
Défis du détournement TCP
  • Prédiction précise des numéros de séquence
  • Synchronisation avec la victime
  • Maintenir la session sans déclencher d'alertes
  • Éviter les mécanismes de détection

Burp Suite - Fonctions clés
  • Proxy d'interception
  • Repeater pour modifier et renvoyer des requêtes
  • Scanner de vulnérabilités
  • Intruder pour attaques par brute force
  • Comparateur de réponses

Démonstration: Utilisation de Burp Suite

  1. Configurer le navigateur pour utiliser le proxy Burp
  2. Activer l'interception des requêtes
  3. Modifier les cookies de session dans une requête
  4. Renforcer les requêtes pour élever les privilèges
  5. Utiliser le Scanner pour trouver des vulnérabilités

Fonctionnalités avancées

  • Session Handling Rules: Automatisation des manipulations
  • Macros: Réexécution de séquences
  • Collaborator: Détection des vulnérabilités hors-bande
  • Extensions BApp: Fonctionnalités supplémentaires
Considérations éthiques

Ces outils ne doivent être utilisés qu'avec autorisation explicite. Leur utilisation non autorisée est illégale et contraire à l'éthique.

5. Contre-mesures

Protections pour utilisateurs

  • HTTPS partout: Toujours utiliser des connexions chiffrées
  • VPN: Sur les réseaux publics non fiables
  • Déconnexion: Après chaque session sensible
  • Mises à jour: Navigateurs et OS à jour
  • Sensibilisation: Reconnaître les signes d'attaques

Protections pour développeurs

  • Cookies Secure, HttpOnly et SameSite
  • Renouvellement des ID de session après login
  • Validation des tokens CSRF
  • Limitation de durée des sessions
  • Enregistrement des activités suspectes

Protections réseau

  • Chiffrement: TLS/SSL pour tout le trafic
  • Segmentation: Isoler les segments réseau sensibles
  • Détection d'intrusion: Systèmes IDS/IPS
  • Authentification forte: MFA pour tous les accès
  • Contrôle ARP: Protection contre ARP spoofing

Bonnes pratiques de sécurité

  • Implémenter HSTS (HTTP Strict Transport Security)
  • Utiliser des cookies de session sécurisés
  • Valider l'origine des requêtes
  • Limiter la durée des sessions
  • Mettre en œuvre des mécanismes de réauthentification

Programme de protection complet

  1. Audit régulier des mécanismes de session
  2. Formation des développeurs aux bonnes pratiques
  3. Tests d'intrusion et évaluations de sécurité
  4. Surveillance continue des sessions
  5. Plan de réponse aux incidents de session
