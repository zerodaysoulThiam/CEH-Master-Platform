1. Concepts du sniffing

Qu'est-ce que le sniffing réseau?

Le sniffing réseau est une technique d'écoute clandestine des communications réseau. Il permet de capturer et d'analyser le trafic réseau transitant par une interface.

Types de sniffing
  • Passif: Écoute sans modification du trafic (hub, port SPAN)
  • Actif: Manipulation du trafic pour capturer plus de données (ARP spoofing)

Modes d'interface réseau

Mode non promiscuité Seul le trafic destiné à l'hôte est traité
Mode promiscuité Toutes les trames sont capturées, quelle que soit la destination
Mode monitor Capture sans fil des paquets (Wi-Fi)

Protocoles vulnérables au sniffing

  • HTTP: Transmet les données en clair
  • FTP: Identifiants non chiffrés
  • SMTP: Emails non chiffrés
  • Telnet: Commande en clair
  • POP3/IMAP: Emails non chiffrés

Architecture réseau et sniffing

Dans les réseaux commutés, le sniffing nécessite des techniques actives comme l'ARP spoofing

2. Attaques MAC

Table MAC et commutation

Les commutateurs (switches) utilisent des tables MAC pour diriger le trafic vers les ports appropriés. Les attaques MAC exploitent ce mécanisme :

Attaque par inondation MAC

  1. L'attaquant génère des trames avec des adresses MAC sources aléatoires
  2. La table MAC du switch se remplit jusqu'à sa capacité
  3. Le switch passe en mode "fail-open" (hub)
  4. Tout le trafic est diffusé sur tous les ports
  5. L'attaquant peut maintenant sniffer tout le trafic
# Créer une inondation MAC avec macof
macof -i eth0 -n 100000

Attaque par usurpation MAC

L'attaquant change son adresse MAC pour :

  • Contourner le filtrage MAC
  • Imiter un appareil légitime
  • Détourner des sessions réseau
Changer l'adresse MAC sous Linux
# Désactiver l'interface
ifconfig eth0 down

# Changer l'adresse MAC
ifconfig eth0 hw ether 00:11:22:33:44:55

# Réactiver l'interface
ifconfig eth0 up

Contre-mesures

  • Port Security sur les switches
  • Limitation du nombre d'adresses MAC par port
  • Détection des changements MAC fréquents

3. Attaques DHCP

Processus DHCP

  1. DHCP Discover (Client → Broadcast)
  2. DHCP Offer (Serveur → Client)
  3. DHCP Request (Client → Serveur)
  4. DHCP ACK (Serveur → Client)

Types d'attaques DHCP

Attaque Description
DHCP Starvation Épuise le pool d'adresses IP disponibles
Rogue DHCP Server Serveur malveillant distribuant de fausses configurations
DHCP Snooping Attaque visant à capturer les informations DHCP
Créer un serveur DHCP rogue
# Utiliser dnsmasq comme serveur DHCP malveillant
dnsmasq --interface=eth0 --dhcp-range=192.168.1.100,192.168.1.200,12h --dhcp-option=3,192.168.1.1 --dhcp-option=6,8.8.8.8

Contre-mesures DHCP

DHCP Snooping

Filtrage des messages DHCP non autorisés

Port Security

Limite les adresses MAC par port

Authentification 802.1X

Contrôle d'accès réseau

4. Empoisonnement ARP

Fonctionnement d'ARP

Le protocole ARP (Address Resolution Protocol) résout les adresses IP en adresses MAC :

  1. Broadcast ARP Request: "Qui a l'IP X.X.X.X?"
  2. Unicast ARP Reply: "L'IP X.X.X.X est à MAC AA:BB:CC:DD:EE:FF"

ARP Spoofing/Poisoning

L'attaquant envoie de fausses réponses ARP pour :

  • Rediriger le trafic vers sa machine
  • Réaliser des attaques Man-in-the-Middle
  • Intercepter ou modifier les communications
# Utiliser arpspoof pour l'empoisonnement ARP
arpspoof -i eth0 -t 192.168.1.10 -r 192.168.1.1

Processus d'attaque

  1. Attaquant → Victime: "192.168.1.1 est à [MAC Attaquant]"
  2. Attaquant → Routeur: "192.168.1.10 est à [MAC Attaquant]"
  3. Le trafic entre la victime et le routeur passe par l'attaquant

Dangers de l'ARP poisoning

  • Interception des mots de passe non chiffrés
  • Détournement de session
  • Attaques par déni de service
  • Modification du trafic en temps réel
  • Écoute des communications chiffrées (SSL stripping)

5. Attaques par spoofing

Types de spoofing

Type Description
IP Spoofing Usurpation d'adresse IP source
MAC Spoofing Usurpation d'adresse MAC
DNS Spoofing Fausses réponses DNS
Email Spoofing Usurpation d'expéditeur d'email
Web Spoofing Copie de sites légitimes

IP Spoofing

Techniques d'usurpation d'IP :

  • Direct Spoofing: Envoi de paquets avec IP source falsifiée
  • Source Routing: Spécification du chemin réseau
  • Blind Spoofing: Sans voir les réponses

Création de paquets spoofés

from scapy.all import *

# Créer un paquet ICMP avec IP source spoofée
packet = IP(src="192.168.1.100", dst="8.8.8.8")/ICMP()/"Payload"

# Envoyer le paquet
send(packet)

Utilisations du spoofing

  • Contourner les ACL basées sur IP
  • Attaques DDoS (amplification)
  • Masquer la véritable source d'attaque
  • Détournement de session TCP

6. Empoisonnement DNS

Fonctionnement du DNS

Le DNS (Domain Name System) traduit les noms de domaine en adresses IP :

  1. Requête DNS vers le résolveur
  2. Recherche récursive ou itérative
  3. Réponse avec l'adresse IP correspondante

DNS Poisoning

L'attaquant corrompt le cache DNS pour :

  • Rediriger les utilisateurs vers des sites malveillants
  • Intercepter les communications
  • Distribuer des malwares
  • Réaliser des attaques de phishing

Techniques d'empoisonnement DNS

  • DNS Spoofing: Fausses réponses DNS
  • DNS Cache Poisoning: Corruption du cache du résolveur
  • DNS Hijacking: Modification des paramètres DNS
  • DNS Tunneling: Transfert de données via des requêtes DNS
# Utiliser dnsspoof pour l'empoisonnement DNS
dnsspoof -i eth0 -f hosts.txt

hosts.txt contient: 192.168.1.100 www.banque.com

Protections DNS

DNSSEC

Signature cryptographique des réponses DNS

DNS over HTTPS (DoH)

Chiffrement des requêtes DNS

Response Rate Limiting

Limite les réponses DNS non sollicitées

7. Outils de sniffing

Analyseurs réseau

Outil Description
Wireshark Analyseur de protocoles complet avec interface graphique
tcpdump Outil en ligne de commande pour la capture de paquets
tshark Version CLI de Wireshark
Ettercap Suite complète pour attaques MITM
BetterCAP Framework moderne pour attaques réseau
Exemple de capture avec tcpdump
# Capturer le trafic sur eth0
tcpdump -i eth0

# Capturer le trafic HTTP
tcpdump -i eth0 port 80

# Sauvegarder dans un fichier pcap
tcpdump -i eth0 -w capture.pcap

Outils spécialisés

  • dsniff: Collection d'outils pour sniffing (arpspoof, dnsspoof)
  • SSLStrip: Dégradation des connexions HTTPS en HTTP
  • Driftnet: Capture d'images transitant sur le réseau
  • NetworkMiner: Analyse forensique de captures réseau
  • Scapy: Puissante bibliothèque Python pour manipulation de paquets

Filtres Wireshark essentiels

http                  # Trafic HTTP
dns                   # Requêtes DNS
tcp.port == 443       # Trafic sur port 443
ip.addr == 192.168.1.1 # Trafic depuis/vers IP spécifique
arp                   # Trames ARP
tcp.flags.syn == 1    # Paquets SYN
!(arp or dns)         # Exclure ARP et DNS

8. Contre-mesures

Protections réseau

  • Chiffrement: HTTPS, SSH, VPN, TLS
  • Commutateurs sécurisés: DHCP snooping, ARP inspection
  • Segmentation réseau: VLANs, sous-réseaux
  • Authentification forte: 802.1X, EAP
  • Détection d'intrusion: IDS/IPS

Protocoles sécurisés

Protocole Utilisation
HTTPS Navigation web sécurisée
SSH Accès distant sécurisé
SFTP/SCP Transfert de fichiers sécurisé
IPSec Chiffrement au niveau IP
DNSSEC Sécurisation des requêtes DNS

Configurations spécifiques

Protection ARP
  • ARP Inspection (DAI) sur les commutateurs
  • ARP Static Binding (table ARP statique)
  • Détection des changements ARP fréquents
Protection DHCP
  • DHCP Snooping
  • Limitation du taux de requêtes DHCP
  • Authentification des serveurs DHCP

Bonnes pratiques

  • Utiliser des protocoles chiffrés partout où possible
  • Mettre en place un VPN pour les accès distants
  • Segmenter le réseau en zones de sécurité
  • Configurer correctement les commutateurs
  • Mettre à jour régulièrement les équipements

9. Techniques de détection

Détection de sniffing

  • Détection de mode promiscuité: Envoyer des paquets avec adresse MAC de destination invalide et détecter les réponses
  • Analyse des temps de réponse: Les interfaces en mode promiscuité peuvent répondre plus lentement
  • Détection ARP: Surveiller les anomalies dans les tables ARP
  • Analyse DNS inverse: Tester si les adresses IP répondent aux requêtes DNS inversées
Détecter le mode promiscuité avec nmap
# Scanner les hôtes en mode promiscuité
nmap --script=sniffer-detect 192.168.1.0/24

Outils de détection

Outil Fonction
ARPWatch Surveillance des changements ARP
XArp Détection avancée des attaques ARP
Sniffdet Détection des interfaces en mode promiscuité
Wireshark Détection des anomalies réseau
Security Onion Distribution de sécurité pour surveillance réseau
Indicateurs de compromission
  • Paquets ARP inhabituels
  • Trafic DNS anormal
  • Activations répétées du mode promiscuité
  • Augmentation soudaine du trafic réseau

Recommandations finales CEHv13

  1. Toujours chiffrer les données sensibles en transit
  2. Implémenter des contrôles de sécurité sur les commutateurs
  3. Surveiller régulièrement le réseau pour détecter des anomalies
  4. Mettre en place des solutions IDS/IPS
  5. Former les utilisateurs aux risques du sniffing
