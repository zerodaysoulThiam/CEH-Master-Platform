1. Banner Grabbing

Le banner grabbing est une technique qui consiste à récupérer des informations de type "bannière" émises par un système ou un service réseau. Ces informations comprennent généralement le nom du logiciel, sa version, et parfois des détails sur le système d’exploitation. Elles sont précieuses pour un attaquant car elles permettent d’identifier les services en cours d’exécution et de détecter d’éventuelles vulnérabilités connues afin de planifier une attaque ciblée.

Informations typiquement récupérées

  • Nom et version du système d’exploitation (OS)
  • Logiciels et versions des services (ex : serveur web nginx, serveur FTP)
  • Messages d’erreur et pages web avec extensions révélatrices

Note : Pour se protéger, il est conseillé de désactiver ou modifier les bannières par défaut et de masquer les extensions des pages web afin de limiter les informations accessibles publiquement.

Types de Banner Grabbing

Banner Grabbing Passif

Cette méthode consiste à observer le trafic réseau sans interagir directement avec la cible. Par exemple, en analysant les messages d’erreur, les paquets réseau capturés ou les extensions visibles sur un site web, on peut souvent déduire le type de système ou logiciel utilisé. Cette approche est discrète et moins susceptible d’être détectée.

Banner Grabbing Actif

Ici, l’attaquant envoie des paquets spécifiques à la cible et analyse les réponses pour identifier le système. Chaque système d’exploitation implémente sa pile TCP/IP différemment, ce qui se traduit par des variations dans des valeurs telles que le TTL (Time To Live) ou la taille de la fenêtre TCP. Les ports communs utilisés sont HTTP (80), FTP (21), SMTP (25), entre autres.

Outils courants de Banner Grabbing

  • nmap : Très utilisé pour le scan de ports et la reconnaissance OS grâce à l’option -O qui effectue une empreinte automatique du système d’exploitation.
  • netcat : Outil polyvalent permettant d’établir des connexions TCP ou UDP, souvent surnommé le "couteau suisse" du réseau. Exemples :
    • nc 178.128.203.1 21 pour interroger un serveur FTP sur le port 21 et récupérer sa bannière (ex : Welcome to XXX FTP Server).
    • nc 178.128.203.1 80 suivi d’une requête HTTP comme GET /index.html HTTP/1.0 pour obtenir les en-têtes HTTP, notamment la version du serveur web (ex : Server: Apache/2.4.6 (CentOS)).
    Une variante sécurisée est CryptCat, qui chiffre les échanges.
  • telnet : Protocole de communication textuelle utilisé historiquement pour accéder à distance à une interface en ligne de commande. Il est moins utilisé aujourd’hui (remplacé par SSH pour des raisons de sécurité), mais reste utile pour le banner grabbing. Exemple :
    • telnet 23 pour se connecter au port 23 et recevoir la bannière du service si le port est ouvert.
    • La réponse peut ressembler à : Server: Microsoft-IIS/5.0 Date: Fri, 14 Aug 2009 1:14:42 GMT Content-Length:340 Content-Type: text/html

Recommandations pour la sécurité

  • Désactiver ou modifier les bannières par défaut pour ne pas divulguer la version des logiciels.
  • Utiliser des pare-feux pour filtrer les connexions non autorisées aux ports sensibles.
  • Mettre à jour régulièrement les systèmes et services pour corriger les vulnérabilités connues.
  • Surveiller les logs pour détecter toute activité suspecte liée à des tentatives de banner grabbing.

Le banner grabbing est une étape clé de la phase de reconnaissance dans un test d’intrusion (pentest). Comprendre cette technique permet à la fois de mieux détecter les tentatives d’attaque et d’améliorer la sécurisation des systèmes.

2. Contourner les IDS (Systèmes de Détection d'Intrusion) et les Firewalls

Les IDS et firewalls sont des dispositifs clés pour la sécurité réseau. Ils surveillent le trafic et bloquent les attaques connues, mais il existe des techniques pour les contourner, souvent utilisées en tests d'intrusion ou attaques sophistiquées. Voici les principales méthodes et leurs principes.

Fragmentation des paquets (Packet Fragmentation)

Aussi appelée IP fragment scanning, cette technique consiste à découper un paquet TCP en plusieurs fragments plus petits. Le serveur cible reconstitue ensuite les fragments pour traiter la requête complète.

  • Les IDS ont souvent du mal à analyser correctement les paquets fragmentés car cela demande beaucoup de ressources.
  • Tout type de datagramme IP peut être fragmenté : TCP, UDP, ICMP, etc.
  • La fragmentation est aussi utilisée dans la session splicing pour contourner l’analyse HTTP.

Exemples d’outils :

  • Nmap : option -f pour fragmenter les paquets (ex : nmap -f 192.168.1.1), ou --mtu pour définir la taille max des fragments.
  • fragroute : intercepte et modifie le trafic sortant pour envoyer des paquets fragmentés.

Routage source (Source Routing)

Le routage source, ou path addressing, permet de spécifier un chemin précis pour un paquet IP, contournant les routes standard (souvent utilisées par les firewalls). En modifiant le champ Options IP, un attaquant peut forcer un paquet à emprunter un chemin spécifique.

Important : cette technique est presque toujours bloquée par les équipements modernes.

Exemples avec Nmap :

  • Routage source lâche (Loose routing) : le paquet passe par des points spécifiés, mais peut aussi emprunter d’autres routes (ex : --ip-options "L 192.168.0.7 192.168.30.9").
  • Routage source strict (Strict routing) : chaque saut doit être indiqué (ex : --ip-options "S 192.168.0.7 192.168.0.9 ... 192.168.30.9").

Scan leurre (IP Decoy)

Technique où des paquets sont envoyés depuis la machine d’attaque, mais avec des adresses IP source falsifiées (leurres). Cela complique la détection d’un scan car le trafic semble provenir de plusieurs sources.

Avantages : brouille la détection des scans de ports.

Inconvénients : n’apporte aucune information supplémentaire et ralentit le scan.

Exemple Nmap : nmap -D leurre1,leurre2,ME,leurre3 ... cible ou -D RND:10 pour 10 leurres aléatoires.

Usurpation d'adresse IP (IP Spoofing)

Modifier le champ adresse source dans le paquet IP pour faire croire qu’il vient d’une autre machine. Très utilisée pour des attaques DoS, car les réponses retournent à l'adresse usurpée, pas à l’attaquant.

Outils :

  • hping3 cible -a ip_usurpée
  • nmap cible -S ip_usurpée

Détection de l’usurpation IP

  • TTL direct : Le TTL du paquet en réponse diffère de celui attendu si usurpé.
  • Numéro d’identification IP (IPID) : Valeur incrémentée par le système, incohérente si usurpation.
  • Contrôle du flux TCP : Analyse de la fenêtre de réception TCP. Les paquets usurpés ne respectent pas la taille de fenêtre et provoquent des anomalies.

Contre-mesures contre l’usurpation IP

  • Utilisation du chiffrement (SSL/TLS, VPN) pour empêcher la manipulation.
  • Éviter la confiance basée sur l’adresse IP (authentification forte).
  • Filtrage d’entrée (ingress filtering) pour bloquer le trafic entrant non légitime.
  • Filtrage de sortie (egress filtering) pour bloquer l’envoi de paquets usurpés depuis l’intérieur.
  • Utilisation de numéros de séquence TCP initiaux aléatoires (ISN).

Chiffrement (Encryption)

Le chiffrement des communications (ex : SSL/TLS, SSH) empêche l’inspection du contenu des paquets par les IDS, ce qui est une méthode efficace d’évasion.

Pour contourner cela, les défenses avancées utilisent des attaques Man-in-the-Middle (MITM) ou des systèmes d’analyse comportementale du trafic (Network Traffic Analysis, NTA), qui analysent les métadonnées plutôt que le contenu chiffré.

Serveurs Proxy

Les proxies sont des intermédiaires qui relaient les requêtes entre clients et serveurs, permettant d’anonymiser l’origine ou de filtrer le contenu.

Usage des défenseurs :

  • Protéger le réseau interne et masquer les adresses IP.
  • Anonymiser la navigation (ex : VPN, Tor).
  • Filtrer les contenus indésirables et économiser la bande passante.

Usage des attaquants :

  • Dissimuler la source des attaques.
  • Accéder à distance à des réseaux internes via un proxy compromis.
  • Réaliser des attaques MITM en redirigeant le trafic.

Chaînage de proxies (Proxy chaining)

Utilisation de plusieurs proxies en série pour augmenter l’anonymat (ex : client <-- proxy1 <-- proxy2 <-- cible).

Outil notable : ProxyChains, qui permet de configurer facilement ce chaînage sous Linux.

Anonymiseurs et réseaux anonymes

Les anonymiseurs protègent la vie privée en cachant l’adresse IP réelle et en chiffrant le trafic.

  • Tor : réseau distribué de relais, offrant un fort anonymat via plusieurs couches de chiffrement. Utilisé pour contourner la censure et préserver la confidentialité.
  • I2P : alternative pair-à-pair à Tor, centrée sur la confidentialité dans les communications internes au réseau.
  • OS spécialisés comme Tails ou Whonix renforcent la protection contre la surveillance.

En résumé : contourner les IDS et firewalls demande souvent d’exploiter des faiblesses dans la gestion des paquets, l’architecture réseau, ou d’utiliser des outils d’anonymisation et de chiffrement. La défense moderne combine filtrage strict, analyse comportementale et chiffrement pour limiter ces contournements.

Pour approfondir, consulter les ressources officielles telles que le manuel Nmap sur la détection IDS ou les chapitres spécialisés sur l’évasion IDS et l’évasion firewall.

3. Vue d’ensemble du scanning de réseaux

Le scanning de réseau est une phase cruciale dans le processus de reconnaissance en cybersécurité. Il s’agit d’obtenir des informations détaillées sur les hôtes, les ports et les services actifs dans un réseau cible. Cette étape permet de détecter les vulnérabilités des canaux de communication et de préparer un plan d’attaque. Toutefois, les administrateurs systèmes utilisent également ces techniques pour surveiller la disponibilité des services, vérifier les politiques de sécurité et assurer une meilleure gestion de l’infrastructure.

Différentes techniques et outils permettent d’identifier les adresses IP actives, les ports ouverts, les services associés et leurs versions. Il est même possible de créer des paquets personnalisés grâce à des outils comme :

  • Colasoft Packet Builder
  • NetScanTools Pro
  • Packeth

Types de scanning

  • Scan de ports : identifie les ports ouverts et les services associés
  • Scan réseau : détecte les adresses IP disponibles sur le réseau
  • Scan de vulnérabilités : découvre la présence de failles connues dans les services

Scanning dans les réseaux IPv6

Avec l’adoption croissante d’IPv6, le scanning devient plus complexe. Le grand nombre d’adresses possibles rend les balayages (sweeps) moins efficaces. Il est donc souvent nécessaire de recueillir les adresses IPv6 à partir de journaux, d’e-mails ou de captures de trafic. La détection des hôtes prend également plus de temps en raison de la taille accrue des sous-réseaux.

Cartographie et topologie réseau

La cartographie d’un réseau consiste à dessiner une vue d’ensemble de la structure du réseau cible, en indiquant les dispositifs présents (pare-feux, routeurs, commutateurs, IDS/IPS, etc.). Cela aide à comprendre les points faibles et les chemins possibles pour une intrusion. Les outils comme Zenmap (interface graphique de Nmap) permettent d’automatiser cette tâche.

Exemple : dans un lieu public (café), vous pouvez utiliser ipconfig (Windows) ou ifconfig (Linux/macOS) pour identifier votre IP et le sous-réseau. Ensuite, un scan Nmap permet de découvrir les autres hôtes présents et d’établir une carte du réseau.

Conseils et pratiques

  • Avant tout scan, s’assurer d’avoir l’autorisation (scan non autorisé = activité illégale).
  • Privilégier les scans silencieux pour éviter d’être détecté par un IDS/IPS (ex : option -sS dans Nmap).
  • Analyser les résultats pour détecter des services vulnérables (version obsolète, port inhabituel ouvert, etc.).

4. Techniques de Scanning en Cybersécurité

En cybersécurité, le scanning est une phase cruciale du processus de reconnaissance. Il permet d'identifier les hôtes actifs sur un réseau (découverte d’hôtes) et les ports ouverts (scanning de ports) pour mieux comprendre la surface d’attaque.

Objectifs du scanning

  • Identifier les ports ouverts sur une machine (scanning de ports)
  • Déterminer quels hôtes sont actifs sur un réseau (découverte d’hôtes)
  • Collecter des informations sur le système, les services et leur état

💡 Les mêmes techniques peuvent être utilisées pour les deux types de scanning, mais avec des outils ou options différentes.

Protocoles et techniques utilisés

1. Scanning ICMP (couche réseau)

L’ICMP est utilisé pour vérifier si un hôte est actif via des messages de type "echo". Ce type de scan est aussi appelé ping scan.

  • Utilisé principalement pour la découverte d'hôtes
  • Commandes typiques dans Nmap : -PE, -PP, -PM
  • ❗ Les pare-feu bloquent souvent les requêtes ICMP entrantes
  • Ne fournit pas d'informations sur les ports

Types de paquets ICMP utilisés

  • Echo Request (type 8) : réponse attendue type 0 (echo reply)
  • Timestamp (type 13) : vérifie le temps de réponse
  • Address Mask (type 17) : demande le masque de sous-réseau

Balayage ICMP (ICMP sweep)

Permet de scanner un ensemble d'adresses IP pour repérer les hôtes actifs.

Exemples :

  • nmap -sn 192.168.0.0/24 : ping tous les hôtes du sous-réseau
  • nmap -sn 192.168.0.1-30 : ping de l'adresse .1 à .30

2. Scanning TCP (couche transport)

Scan TCP Connect (pleinement ouvert)

  • Utilise la fonction système connect()
  • Effectue un handshake TCP complet
  • Avantage : fiable, pas besoin de privilèges root
  • Inconvénient : facilement détectable
  • Commande : nmap -sT <cible>

Scan SYN (semi-ouvert ou furtif)

  • Arrête la connexion après l'envoi du SYN-ACK
  • Par défaut dans Nmap : -sS
  • Avantage : rapide et discret
  • Inconvénient : nécessite des privilèges root
  • Commandes : nmap -sS, hping3 -S

Scans RFC 793 (sans SYN/ACK/RST)

Utilisent des combinaisons de drapeaux TCP non standard. Très discrets mais parfois inefficaces contre des systèmes non conformes.

Scan XMAS
  • Envoie un paquet avec les flags FIN, URG, PUSH
  • nmap -sX ou hping3 -F -P -U
Scan NULL
  • Sans aucun flag (entête TCP vide)
  • nmap -sN
Scan FIN
  • Envoie uniquement le flag FIN
  • nmap -sF

3. Scanning ACK

Ce type de scan n’identifie pas les ports ouverts, mais sert à détecter la présence de pare-feu et leur comportement.

  • Envoie un paquet avec le flag ACK
  • Utile pour cartographier des pare-feux stateful
  • nmap -sA ou hping3 -A
Classification des réponses ACK

Les analyses ACK interprètent la réponse RST pour déduire l'état du port :

  • TTL < 64 : probablement ouvert
  • Fenêtre TCP (Window) ≠ 0 : probablement ouvert (sur anciens systèmes UNIX/BSD)

4. Scanning IDLE

Méthode avancée et furtive exploitant un hôte "zombie" pour analyser une cible sans exposer l’IP de l’attaquant. Basée sur l’identification IP et les réponses RST.

  • Très difficile à détecter
  • Requiert un zombie ayant un IP ID incrémentiel
  • nmap -sI zombie_ip:port target_ip

Outils recommandés

  • Nmap : le plus complet, supporte tous les types de scan
  • Hping3 : pour des scans plus granulaires au niveau TCP/IP
  • Angry IP Scanner : interface graphique simple pour ICMP sweep
  • Wireshark : pour analyser le trafic et valider les scans

Conseils pratiques

  • 🔐 Toujours effectuer des scans sur des systèmes que vous avez le droit de tester
  • 🕵️‍♂️ Alternez entre scans furtifs (SYN, XMAS, NULL) et complets (TCP connect) selon l’objectif
  • 📉 Privilégiez les scans ICMP en pré-scanning pour éliminer les hôtes inactifs
  • ⏱️ Combinez les options de Nmap pour gagner en efficacité (e.g., -Pn pour ignorer le ping initial)

5. Outils de scan réseau

Nmap (Network Mapper)

Nmap est un outil puissant d’analyse de réseau, utilisé pour détecter les hôtes actifs, les ports ouverts, les systèmes d’exploitation, les services, les pare-feux, et bien plus. Il envoie des paquets spécialement conçus et analyse les réponses pour collecter des informations sur les cibles.

Fonctionnalités principales

  • Découverte d’hôtes et analyse de ports (TCP/UDP).
  • Détection du système d’exploitation et des services (version, bannières).
  • Utilisation de scripts personnalisés grâce au NSE (Nmap Scripting Engine) pour détecter ou exploiter des vulnérabilités.
  • Intégration d’outils complémentaires :
    • Ncat – Équivalent netcat, pour lire/écrire des données via le réseau.
    • ndiff – Pour comparer les résultats de scans.
    • nping – Générateur de paquets et outil d’analyse de latence.

Phases d'un scan avec Nmap

  1. Pré-scan des scripts NSE
  2. Énumération des cibles (résolution DNS, notation CIDR)
  3. Découverte d’hôtes (ping scan)
  4. Résolution DNS inversée
  5. Scan des ports (états : ouvert, fermé, filtré)
  6. Détection de version des services
  7. Détection du système d’exploitation
  8. Traceroute (chemin réseau)
  9. Scan par scripts NSE
  10. Affichage des résultats (format texte, XML, greppable...)
  11. Post-scan (scripts de rapport)

Options courantes de Nmap

  • -n : pas de résolution DNS
  • -A : active OS detection, version, scripts et traceroute
  • --script : exécute des scripts NSE personnalisés
  • -T0 à -T5 : vitesse du scan (de paranoïaque à insensé)
  • -sS, -sT, -sU, -sV : différents types de scan (SYN, connect, UDP, version)
  • -Pn : scan sans ping (considère toutes les cibles comme vivantes)
  • -O : détection du système d’exploitation
  • -p- : scanner tous les ports (1 à 65535)
  • -oX, -oG : export en XML ou format lisible par grep

Spécification des cibles

On peut spécifier une ou plusieurs cibles :

  • Adresse IP directe : nmap 192.168.1.1
  • Nom de domaine : nmap exemple.com
  • Plage CIDR : 192.168.1.0/24
  • Plage personnalisée : 192.168.0-255.1-254
  • Fichier de cibles : nmap -iL liste.txt
  • Cibles multiples : nmap cible1.com, cible2.com

💡 Nmap est souvent utilisé dans les films comme Matrix Reloaded ! C’est un outil emblématique de la cybersécurité.

Hping

Hping est un scanner de ports plus bas niveau que Nmap, très utile pour les tests furtifs. Il permet d’envoyer des paquets ICMP, UDP ou TCP personnalisés, ce qui le rend particulièrement adapté à la recherche de vulnérabilités ou à l’évasion des pare-feux.

Différences entre Hping et Nmap

  • Hping est plus discret, mais ne peut scanner qu'une seule IP à la fois.
  • Nmap est plus complet, supporte IPv6 et les scans à large échelle.
  • Hping est souvent utilisé pour l’analyse des séquences TCP ou l’injection personnalisée de paquets.

Commandes utiles avec Hping

  • --tcp-timestamp : tente de deviner la fréquence d’update TCP et l’uptime du système distant
  • -Q : collecte les numéros de séquence TCP
  • -F, -S, -A, etc. : pour définir des flags TCP (FIN, SYN, ACK...)
  • hping3 -1 10.0.1.x : scan ICMP sur un sous-réseau
  • hping3 -9 HTTP -I eth0 : écoute du trafic HTTP sur l’interface réseau

📚 Consultez la documentation officielle pour plus d'exemples.

Applications mobiles de scan réseau

Pour des analyses rapides depuis un téléphone, les outils suivants sont recommandés :

  • IP Scanner (iOS) – Scanner simple pour voir les appareils sur le réseau local
  • Fing (iOS / Android) – Très populaire, détecte les appareils, les ports ouverts, analyse la bande passante

📱 Les outils mobiles sont idéals pour des audits de réseau rapides, en déplacement ou dans un cadre non professionnel.

Modèle TCP/IP et Fondamentaux Réseau

Connexion TCP : L'Établissement de la Communication

Trois Étapes du Handshake (Three-Way Handshake)

Pour établir une connexion TCP, trois paquets sont échangés entre le client et le serveur :

  1. Client → Serveur : envoie un paquet SYN
  2. Serveur → Client : répond avec SYN-ACK
  3. Client → Serveur : confirme avec ACK

Une fois la connexion établie, chaque paquet envoyé contiendra le drapeau ACK.

Fermeture d'une Connexion TCP

  1. Client → Serveur : envoie FIN
  2. Serveur → Client : répond avec ACK + FIN
  3. Client → Serveur : envoie ACK

Adresses IPv4

L'adresse de boucle locale (localhost) en IPv4 est 127.0.0.1.

Types d'Adresses IPv4

  • Unicast : vers un seul destinataire
  • Multicast : vers un groupe défini d'appareils
  • Broadcast : vers tous les hôtes du réseau

Deux types de broadcast :

  • Broadcast limité : 255.255.255.255 (non relayé par les routeurs)
  • Broadcast dirigé : par exemple 192.168.1.255 pour un sous-réseau /24

Adresses IPv6

  • Utilise 128 bits contre 32 bits pour IPv4
  • Représentée en 8 groupes hexadécimaux séparés par des deux-points
  • Adresse de boucle locale : ::1
  • Les zéros initiaux peuvent être omis : 2001:0db8:0000:0000:0000:0000:0000:0001 devient 2001:db8::1

CIDR (Classless Inter-Domain Routing)

Le CIDR est une méthode de notation pour représenter les adresses IP et les masques de sous-réseau.

Les Flags TCP

Chaque flag est un bit qui décrit une action ou un état de la connexion TCP :

  • SYN : Démarrage de la connexion
  • ACK : Accusé de réception
  • FIN : Terminaison de la connexion
  • RST : Réinitialisation soudaine (utilisée dans certains scans ou attaques)
  • PSH : Données à traiter immédiatement
  • URG : Données urgentes

Sessions TCP/IP

TCP est un protocole à état (stateful) : une connexion est établie avant tout échange de données.

État de la session :

  • Ouverture : Handshake (SYN → SYN-ACK → ACK)
  • Échange de données
  • Fermeture : FIN ou RST

Le Modèle OSI (Open Systems Interconnection)

Modèle théorique en 7 couches qui guide la conception des réseaux :

  1. Physique : Transmission de bits (câbles, signaux)
  2. Liaison de données : Adressage physique (Ethernet, MAC)
  3. Réseau : Routage, IP, ICMP
  4. Transport : Transmission fiable (TCP, UDP)
  5. Session : Gestion des connexions
  6. Présentation : Format des données, chiffrement (SSL, TLS)
  7. Application : Interfaces utilisateurs (HTTP, DNS, FTP, SMTP)

Le Modèle TCP/IP

Modèle pratique à 4 couches utilisé dans l'Internet :

  1. Couche Accès Réseau (Link) : Ethernet, ARP
  2. Couche Internet : IP, ICMP, IGMP
  3. Couche Transport : TCP, UDP
  4. Couche Application : HTTP, FTP, DNS, SSH, SMTP

Comparaison OSI vs TCP/IP

Modèle TCP/IP Protocoles Modèle OSI
Application HTTP, FTP, DNS, SMTP Application, Présentation, Session
Transport TCP, UDP Transport
Internet IP, ICMP, ARP Réseau
Accès réseau Ethernet, Wi-Fi Liaison, Physique

Note : Certains protocoles comme SSL/TLS ne s’intègrent pas parfaitement à l’un ou l’autre modèle, étant situés entre plusieurs couches.
