1. Concepts IDS/IPS, Firewall & Honeypot

Systèmes de défense réseau

Système Fonction
Firewall Filtrage du trafic basé sur des règles prédéfinies
IDS (Système de détection d'intrusion) Surveillance et alerte en cas d'activité suspecte
IPS (Système de prévention d'intrusion) Blocage actif des menaces détectées
Honeypot Système leurre pour attirer et étudier les attaquants
Différence IDS vs IPS
  • IDS: Détection passive (alerte sans bloquer)
  • IPS: Détection active (bloque le trafic malveillant)
  • Les IPS sont placés inline (dans le flux du trafic)
  • Les IDS fonctionnent en mode miroir (hors du flux principal)

Types de Honeypots

  • Low-Interaction: Simule des services basiques
  • High-Interaction: Systèmes complets pour étude approfondie
  • Production: Protège les réseaux réels
  • Research: Collecte de données pour analyse
  • Honeynets: Réseaux entiers de honeypots

Architecture de sécurité typique

Firewall → IPS → IDS → Honeypot

Méthodes de détection

Signature-based

Détection basée sur des modèles connus de menaces

  • Avantage: Détection précise des menaces connues
  • Limite: Inefficace contre les nouvelles menaces
Anomaly-based

Détection basée sur des écarts par rapport au comportement normal

  • Avantage: Détecte les menaces inconnues
  • Limite: Fort taux de faux positifs
Behavior-based

Analyse du comportement des applications et utilisateurs

  • Avantage: Détection des menaces persistantes
  • Limite: Complexe à configurer

3. Contournement des IDS

Techniques fondamentales

Technique Description
Fragmentation IP Division des paquets pour éviter la détection
Encodage Modification de la charge utile pour échapper aux signatures
Trafic chiffré Utilisation de VPN/SSL pour masquer le contenu
Flux lent Envoi de données à très bas débit

Évasion par polymorphisme

Modification continue de la signature du malware pour éviter la détection basée sur les signatures.

  1. Chiffrement de la charge utile
  2. Obfuscation du code
  3. Modification des métadonnées
  4. Techniques de métamorphisme

Techniques avancées

  • Session Splicing: Division des sessions en petits fragments
  • Overlapping Fragments: Fragments qui se chevauchent pour confondre le réassemblement
  • Invalid Packets: Paquets mal formés exploitant les faiblesses de l'IDS
  • Timing Attacks: Exploitation des délais d'analyse

Démonstration: Fragmentation IP

  1. Création d'un paquet malveillant
  2. Division en plusieurs fragments
  3. Modification des offsets IP
  4. Envoi avec des délais variables
  5. Réassemblage côté victime

Détection des IDS

  • Analyse des temps de réponse
  • Envoi de paquets de test spécifiques
  • Détection des patterns de surveillance
  • Analyse du comportement réseau

4. Contournement des Firewalls

Méthodes classiques

Technique Description
Port Hopping Utilisation de ports non standard
Port Forwarding Redirection de ports via des machines compromises
HTTP Tunneling Encapsulation dans le protocole HTTP
DNS Tunneling Utilisation des requêtes DNS pour exfiltrer des données

Démonstration: DNS Tunneling

  1. Configuration d'un serveur DNS malveillant
  2. Encodage des données dans des sous-domaines
  3. Envoi des requêtes via le port 53 (généralement ouvert)
  4. Décodage des données côté serveur

Techniques avancées

  • ICMP Tunneling: Utilisation du protocole ICMP (Ping)
  • FTP Bounce: Exploitation des serveurs FTP pour contourner les restrictions
  • Firewalking Détermination des règles de pare-feu par analyse Application-level Utilisation de services autorisés (HTTP, SMTP)
    Protections contre le contournement
    • Inspection approfondie des paquets (DPI)
    • Analyse comportementale du trafic
    • Limitation des ports ouverts
    • Surveillance des flux sortants

6. Outils de contournement

Outils populaires

Outil Fonctionnalité
Nmap Scan réseau, détection de pare-feux
Metasploit Framework d'exploitation avec modules d'évasion
DNS2TCP Tunnel DNS
Ptunnel Tunnel ICMP
HTTPTunnel Encapsulation HTTP
Nmap - Techniques d'évasion
  • --scan-delay: Délai entre les paquets
  • --data-length: Ajout de données aléatoires
  • --frag: Fragmentation des paquets
  • --source-port: Spécification du port source
  • --spoof-mac: Spoofing d'adresse MAC

Démonstration: Tunnel DNS avec DNS2TCP

  1. Configuration du serveur DNS
  2. Lancement du serveur tunnel: dns2tcpd -f dns2tcpd.conf
  3. Configuration du client: dns2tcpc -z domain.com server_ip
  4. Connexion via SSH sur le tunnel

Fonctionnalités avancées de Metasploit

  • Charges utiles chiffrées
  • Techniques de fragmentation
  • Évasion des antivirus
  • Migration de processus
  • Communication chiffrée
Considérations éthiques

Ces outils ne doivent être utilisés qu'avec autorisation explicite. Leur utilisation non autorisée est illégale.

7. Détection des Honeypots

Techniques de détection

Méthode Description
Analyse de comportement Détection d'anomalies dans les réponses système
Fingerprinting Identification des signatures spécifiques aux honeypots
Délais artificiels Détection des temps de réponse non naturels
Interactions limitées Observation des limites fonctionnelles
Indicateurs communs des honeypots
  • Services avec versions inhabituelles
  • Absence de trafic utilisateur réel
  • Réponses trop "propres" ou standardisées
  • Délais de réponse constants
  • Absence d'activité système normale

Outils de détection

  • Honeypot Detector: Identification des honeypots connus
  • Artillery: Surveillance pour détecter les leurres
  • Wireshark: Analyse des patterns réseau
  • Nmap Scripts: Détection spécifique via NSE

Étapes pour identifier un honeypot

  1. Scan initial des ports et services
  2. Analyse des bannières et versions
  3. Test d'interactions simples
  4. Observation des réponses aux erreurs
  5. Analyse du trafic réseau environnant
  6. Comparaison avec des systèmes réels
Précautions

La détection des honeypots peut alerter les défenseurs. Toujours utiliser des techniques non intrusives et des connexions anonymisées.

8. Contre-mesures

Protections pour IDS/IPS

  • Mises à jour régulières: Maintenir les signatures à jour
  • Détection comportementale: Compléter les signatures par de l'IA
  • Configuration avancée: Réglage fin des seuils de détection
  • Analyse SSL/TLS: Inspection du trafic chiffré
  • Corrélation: Intégration avec SIEM

Protections pour Firewalls

  • Politiques de moindre privilège
  • Inspection approfondie des paquets (DPI)
  • Blocage des tunnels connus (DNS, ICMP)
  • Surveillance des flux sortants
  • Segmentation réseau stricte

Protections pour Honeypots

  • Diversification: Varier les types de honeypots
  • Personnalisation: Adapter les leurres à l'environnement réel
  • Interaction réaliste: Comportements crédibles
  • Intégration: Lier aux systèmes de détection
  • Anonymat: Dissimuler l'origine des honeypots

Stratégies globales

  • Défense en profondeur (multi-couches)
  • Surveillance continue du réseau
  • Tests d'intrusion réguliers
  • Formation des équipes de sécurité
  • Plan de réponse aux incidents

Programme de protection complet

  1. Évaluation des risques et des vulnérabilités
  2. Mise en œuvre de contrôles défensifs multi-couches
  3. Configuration et durcissement des systèmes
  4. Surveillance et analyse continue
  5. Tests réguliers d'efficacité des contrôles
  6. Mise à jour et amélioration continue

Solutions intégrées

XDR (Extended Detection and Response)

Intégration des données de sécurité

SASE (Secure Access Service Edge)

Convergence réseau et sécurité

SOAR (Security Orchestration, Automation and Response)

Automatisation de la réponse
